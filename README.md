# Credit Risk Assessment System – Business Analysis Case Study

## Project Overview
This project presents a Business Analysis case study for a Credit Risk Assessment System designed for financial institutions to evaluate loan applications and reduce default risk.

The objective is to automate the loan risk evaluation process using customer financial data and support loan officers in making data-driven approval decisions.

---

## Business Problem
Financial institutions often face losses due to loan defaults caused by inadequate risk assessment during loan approvals.

The organization requires a system that can:

- Analyze applicant financial data
- Evaluate credit risk
- Provide loan approval recommendations
- Reduce manual decision-making

---

## Business Objectives

- Automate credit risk evaluation
- Improve loan approval decision accuracy
- Reduce loan default risk
- Increase loan processing efficiency

---

## Stakeholders

| Stakeholder | Responsibility |
|-------------|---------------|
| Loan Officer | Reviews applications and approves/rejects loans |
| Risk Analyst | Evaluates customer credit risk |
| Data Analyst | Builds and maintains risk evaluation models |
| IT Team | Implements the system |

---

## Project Scope

### In Scope
- Loan application submission
- Customer financial data collection
- Credit risk score calculation
- Loan approval recommendation system
- Risk evaluation dashboard

### Out of Scope
- Credit bureau integrations
- Loan disbursement processing
- Payment collection systems

## Functional Requirements (Examples)

| ID | Requirement |
|----|-------------|
| FR-1 | System should allow customers to submit loan applications |
| FR-2 | System should collect customer financial data |
| FR-3 | System should calculate credit risk score |
| FR-4 | System should classify customers into risk categories |
| FR-5 | System should display risk score to loan officer |
| FR-6 | System should provide loan approval recommendation |

---

## Non-Functional Requirements

- System response time < 3 seconds
- Secure storage of financial data
- 99% system availability

---

## Business Rules

- High default probability applicants should be flagged as High Risk
- Risk score must be calculated before loan decision
- Loan officers can override system recommendations

---

## Process Workflow

The system workflow follows these steps:

1. Customer submits loan application
2. System collects financial data
3. Risk evaluation system calculates credit score
4. System categorizes applicant risk level
5. Loan officer reviews risk score
6. Loan is approved or rejected
